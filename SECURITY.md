# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x.x   | :white_check_mark: |
| 1.x.x   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this portfolio website, please report it by emailing **ervin.ward@example.com**.

**Please do NOT open a public issue.**

### What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

### Response Time

- You can expect an initial response within **48 hours**
- We will investigate and provide updates within **7 days**
- A fix will be released as soon as possible, depending on severity

### Security Best Practices

This portfolio follows these security practices:

- ✅ No sensitive data stored in client-side code
- ✅ All external links use `rel="noopener noreferrer"`
- ✅ Content Security Policy headers configured
- ✅ Regular dependency updates
- ✅ No inline scripts (CSP compliant)
- ✅ HTTPS enforced via GitHub Pages

Thank you for helping keep this project secure!
